What did we Evaluate?

Edge Presentation

  • Azure Options - Registrar, DNS, Frontdoor, CDN
  • CloudFlare - Registrar, DNS, CDN, SSL
  • Akamai Services - DNS, CDN, SSL
  • Fastly - CDN
  • AWS CloudFront & Route 53 - DNS, CDN
  • Google Cloud - DNS, CDN

How did we evaluate?

  • Collected technical requirements
  • Built prototypes to get feedback
  • Measured value versus cost of predicted change

Azure - Evaluation Outcome Summary

Azure - Registrar, DNS, CDN, WAF, SSL

  • Complex to manage, poorly integrated, mostly partner delivered, weak IAC support
  • DNS Registrar - Partner provided / slow to build
  • DNS Services - Not validated, lacking security
  • FrontDoor - Reliability issues over last 12 months
  • CDN - Semi partner provided / slow to build
  • WAF - Not mature or easy to manage at scale
  • SSL - Partner provided / slow to build

Akamai - Evaluation Outcome Summary

Akamai - DNS, CDN, WAF, and SSL

  • Very complex to manage at scale, expensive, and optimized for single site support, but good IAC support
  • DNS Services - Robust
  • CDN (ION) - Best in class but complex to manage across multiple properties
  • WAF - Best in class
  • SSL - Partner provided / slow to build

Cloudflare - Evaluation Outcome Summary

Cloudflare - Registrar, DNS, CDN, WAF, SSL

  • Cost effective, replaces the most services, best best IAC support
  • DNS Registrar - Can only be transferred after 60 days of ownership
  • DNS Services - Best in class
  • SSL - Provisions instantly
  • CDN - Easy to manage at scale
  • WAF - Easy to manage at scale

Fastly, AWS, Google Cloud

Missing compatible technical capabilities

  • Fastly - Does not provide Domain Registrar or DNS services
  • Azure blocks integration between these services in AWS and Google Cloud

Next Steps - Cloudflare

  1. Start contracts process
  2. Start migration of DNS from UltraDNS
  3. Build out IAC examples for technical teams to use
  4. Apply CDN and SSL termination service to applications via SSL life cycle process (Certificates last 1-2 years)
  5. Migrate domain registrations via life cycle process (Domains normally renew every year)
