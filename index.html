What is Application and Systems Event management?

A methodology and process to monitor, aggregate and make sense of disparate events that occur through the IT Infrastructure and the Services within

What did we Evaluate?

5 Solutions

  • Azure
  • New Relic
  • Splunk
  • Rollbar
  • Datadog

How did we evaluate?

  • Collected technical requirements and use cases
  • Built prototypes to garner feedback
  • Measured and scored upon multiple criteria across the solutions

Evalulation Criteria

  • Full SaaS Events and Logs Collection Platform
  • Easy to Use, Configure, and Loosely Coupled
  • Supports Application and System Logging Contexts
  • ...and More

Azure Evaluation

  • Pro: Supports System and Native Apps logging
  • Pro: Variety of ways to interact with the platform
  • Con: Logging mediums not in parity between Azure hosted apps and the Log Analytics Agent (MMA)
  • Con: Each agent holds its own unique configuration and makes it difficult to manage at scale
  • Con: Logging platform serving as also "The Log Collector"

New Relic Evaluation

  • Pro: Supports System and Native Apps logging
  • Pro: Kubernetes instrumentation is a native integration
  • Pro: Supports basic tagging on data inputs/on ingest
  • Con: Logging capability is not as mature compared to other tools
  • Con: Requires more plugins/dependencies than other tools

Splunk Evaluation

  • Pro: Supports System and Native Apps logging
  • Pro: Market Leader in Log Collection and Aggregation
  • Con: Agent configuration is unwiedly and hard to setup
  • Con: Agent and data inputs config carry unique customization and become expensive to maintain

Rollbar Evaluation

  • Pro: Complimentary tool in nature to further enrich logging experience
  • Pro: Code traceback feature makes root cause id quick
  • Pro: Developed SDK for Salesforce Apex Apps
  • Con: Not intended for system level logging

Datadog Evaluation

  • Pro: Supports System and Native Apps logging
  • Pro: Native apps logging is agentless!
  • Pro: Maturity in logging space
  • Pro: 400+ built-in integration, 55 are related to Azure
  • Con: Instrumentation for Kubernetes Logging carried more dependencies compared to other solutions
